WP REST Proxy Function

Exposes a WordPress REST API call (/wp/v2/…) as a chat‑invokable function, with built‑in parameter mapping and optional cursor pagination.

Function Name

The name by which this function is called from a chat message.

Description

Brief explanation shown in function selectors, e.g. “List all posts with filters and paging.”

Required Parameters

Expression‑language array of parameter keys that must be present. Wrap in ${…}. Example: ${["postId"]}

Default Values

Expression‑language object of default parameter values. Wrap in ${…}. Example: ${{"per_page":10,"status":"publish"}}

HTTP Method

HTTP verb to use (GET, POST, PUT, DELETE, etc.). Defaults to GET.

REST Endpoint

Path under /wp/v2/, for example posts or users/123.

Enable Pagination

When checked, responses include results and nextCursor for cursor‑based paging.

Function Parameters

Define metadata for each parameter (type, description, enum, etc.) via a sub‑flow. Use this to generate JSON‑Schema for validation and UI hints.
